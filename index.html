roboclone

Clone FRC robot projects from Git and configure them for use with Eclipse

View on GitHub Download .zip Download .tar.gz

The Problem

roboclone was created to solve the problem of there not being any good way to import code cloned from Git repositories into Eclipse. Typically your options are:

  • Check your IDE metadata files into Git (eww!)
  • Deal with not having a proper WPILib development environment
  • Do the crazy 8-step hack someone figured out once and put on your internal wiki

None of these solutions were good enough for me, so I created roboclone. With roboclone, cloning a repository from Git and having it ready for use in Eclipse is as easy as roboclone <YOUR_REPO_HERE>.

What Does roboclone Do?

roboclone knows what files are necessary to have a good setup in Eclipse. When you use roboclone to clone a repository, it will perform the following steps:

  1. Clone the repository using the system's Git executable (make sure Git is in the system path!)
  2. Figure out the name of your project, and generate the necessary IDE metadata files
  3. Write the IDE metadata files if there are none in the project folder (unless the user forces overwriting with the --force switch

Examples

Command Description
roboclone <REPO> Clones the repository and writes metadata files if they don't already exist
roboclone <REPO> /tmp/my-project Same as above, but the project will be cloned to /tmp/my-project
roboclone --gitargs="--depth=1" <REPO> Clone the repository and pass --depth=1 to git
roboclone --force=true <REPO> Clone the repository and overwrite any existing IDE metadata files
